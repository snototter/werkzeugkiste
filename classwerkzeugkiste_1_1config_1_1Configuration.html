bool werkzeugkiste::config::Configuration::AdjustRelativePaths(std::string_view base_path, const std::vector<std::string_view>& parameters) pure virtual

Adjusts the given parameters to hold either an absolute file path, or the result of "base_path / <param>" if they initially held a relative file path.

Parameters
base_path Base path to be prepended to relative file paths.
parameters A list of parameter names / patterns. The wildcard '*' is also supported. For example, valid names are: "my-param", "files.video1", etc. Valid patterns would be "*path", "some.nested.*.filename", etc.
Returns True if any parameter has been adjusted.

void werkzeugkiste::config::Configuration::LoadNestedTOMLConfiguration(std::string_view key) pure virtual

Loads a nested TOML configuration.

Parameters
key Parameter name (fully-qualified TOML path) which holds the file name of the nested TOML configuration (must be of type string)

For example, if your configuration had a field "storage", which should be defined in a separate (e.g. machine-dependent) configuration file, the "main" config could define it as storage = "path/to/conf.toml" This function will then load this TOML and replace storage by the loaded configuration. Suppose that conf.toml defines location = ... and duration = .... Then, after loading, you can access these as "storage.location" and "storage.duration".
